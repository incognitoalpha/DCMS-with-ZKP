# DCMS - Zero-Knowledge Proof Implementation Guide

## Project Overview

DCMS (Decentralized Cottage Management System) is a blockchain project featuring Zero-Knowledge Proofs (ZKP) for privacy-preserving bookings, governance, and reputation tracking.

The application is completely ZKP-based for bookings and governance to ensure privacy.

---

## ZKP Architecture

The project implements a full ZKP-based architecture using Groth16 proofs generated in the browser and verified on-chain.

### Implementation Core

#### 1. Private Bookings
- **Goal**: Hide booker identity and details while preventing double-booking.
- **Implementation**: Bookings are stored as commitments: `hash(secret, resourceId, startTime, endTime)`.
- **Integrity**: Double-booking is checked on-chain via overlapping times, and double-use of commitments is prevented via nullifiers.
- **Verification**: The contract verifies the proof before saving the booking.

#### 2. Anonymous Governance
- **Goal**: Vote on proposals without revealing the voter's identity.
- **Implementation**: Users register an identity commitment. Voting requires proving membership in the identity set without revealing which identity is voting.
- **Nullifiers**: Double-voting is prevented via nullifiers specific to the proposal.

#### 3. Private Reputation
- **Goal**: Prove reputation thresholds without revealing the exact score.
- **Implementation**: The circuit proves that the user's secret score is greater than a specific threshold.

## File Structure

### Smart Contracts

| File | Purpose |
|------|---------|
| `contracts/contracts/DCMS.sol` | Main contract — handles resources, ZK identities, bookings, governance, and reputation |
| `contracts/contracts/*Verifier.sol` | Groth16 verifier contracts generated by snarkjs |

### ZK Circuits

| File | Purpose |
|------|---------|
| `contracts/circuits/booking.circom` | Proves a valid booking creation/cancellation |
| `contracts/circuits/voting.circom` | Proves group membership + nullifier for voting |
| `contracts/circuits/reputation.circom` | Proves score > threshold |

## Contract Function Signatures

### Resource Management (Admin)
```solidity
function addResource(string calldata name, string calldata description, uint256 pricePerHour) external onlyAdmin returns (uint256);
function setResourceActive(uint256 resourceId, bool active) external onlyAdmin;
```

### ZK Functions
```solidity
function registerIdentity(uint256 commitment) external;

function vote(
    uint256 proposalId,
    bool support,
    uint256 identityCommitment,
    uint256 nullifier,
    uint256[2] calldata a,
    uint256[2][2] calldata b,
    uint256[2] calldata c
) external;

function bookResource(
    uint256 commitment,
    uint256 nullifier,
    uint256 resourceId,
    uint256 startTime,
    uint256 endTime,
    uint256[2] calldata a,
    uint256[2][2] calldata b,
    uint256[2] calldata c
) external payable;
```
